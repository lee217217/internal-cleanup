部署說明

  1. 將整個專案上傳到 Netlify。
  2. 在 Netlify env 設定 PPLX_API_KEY。
  3. 如有內部修補 API，再設定 IMAGE_EDIT_API_URL 與 IMAGE_EDIT_API_KEY。
  4. 前端只呼叫 /.netlify/functions/*，金鑰不會暴露在瀏覽器。
Perplexity 的角色

Perplexity API 已支援圖像附件分析，可用 base64 data URI 送圖。[web:62][web:65] 但公開文件目前重點是圖像理解與多模態輸入，不是明確的像素級修補 API，因此這個方案把它放在分析/提示層更穩妥。[web:62][web:65][web:61]
